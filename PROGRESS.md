# Mynecraft — Build Progress  *(by Hassan Nazir)*

## Stabilization pass (before Phase 11) ✅
Bugs found and fixed:
1. **AABB collision clamped to the first overlapping cell, not the most restrictive** (`Player.moveAxis`) — with several overlapped cells the player could clamp to the wrong plane and clip beside steps. Now scans all cells and takes the min/max candidate per travel direction.
2. **Leaves rendered in the transparent pass** (depth-write off, double-sided) — caused see-through canopies and sort artifacts against water/glass. Switched to opaque "fast" leaves (full alpha, occluding), which also culls more faces.
3. **Fresh worlds could spawn underwater** — spawn used `heightAt(0,0)` unconditionally. Added a spiral search for the nearest dry column above sea level.

Audited and confirmed OK: cross-chunk-border face culling (world-coord neighbor lookups + neighbor dirty-marking on border edits), re-mesh after edits (chunk + border neighbors flagged), DDA raycast face normals, placement-inside-player rejection.

All phases complete. `npm run dev` → http://localhost:5173. Zero TypeScript errors (`npm run typecheck`), zero console errors.

## WASD direction fix ✅
- **Movement was mirrored on X as you turned.** The yaw-rotation of the WASD input vector had both cross-terms negated (`vx = mx·cos − mz·sin`, `vz = mz·cos + mx·sin`). Correct rotation for camera forward `(−sin,−cos)` / right `(cos,−sin)` with W=`mz−1`, D=`mx+1` is `vx = mx·cos + mz·sin`, `vz = mz·cos − mx·sin`. It happened to be correct only near yaw≈0 (sin≈0), which is why it felt fine facing one way and drifted wrong after looking around.
- **Verify (scripted):** W·lookDir = 1.0, S·lookDir = −1.0, D·cameraRight = 1.0 at yaw 0/45/90/180/270 — exact at every angle.

## Controls & interaction fix pass ✅
1. **One-shot input double/stale firing:** `justPressed`/`buttonJustPressed` were cleared once per rendered frame, but the fixed-timestep loop can run twice per frame — a single Space press could read as a double-tap (random flight toggles in creative), Q dropped two items, melee swung twice. Now cleared at the end of each fixed tick; input buffered while paused is discarded (`clearTransient`), so nothing fires and the camera doesn't jerk on resume.
2. **Phantom fall damage via water:** fall distance kept accumulating while falling into water, then applied on the first solid landing when wading ashore. Water (and flight) now zero fall distance.
3. **No swimming:** deep water was inescapable. Added water physics — body submerged: hold Space to rise (cap 3.5), slow sink (cap −4), reduced gravity, 0.55× move speed.
4. **Unhandled pointer-lock failures:** re-locking during the browser's post-Esc cooldown threw unhandled rejections and could leave a paused-but-menuless state. All lock requests now go through `Input.requestLock()` with a fallback that opens the pause menu.
5. **Food blocked crafting tables/furnaces:** holding food swallowed right-click entirely; interactive block targets now take priority over eating.
6. **F3 misfiring:** the debug toggle polled `justPressed` from the render loop, where ticks may already have consumed it — moved to a dedicated keydown listener (also works while paused).

## Phase 24 — Brewing ✅
- **Simplified vs. vanilla:** no Nether-exclusive ingredients (blaze powder / nether wart don't exist yet — the Nether is Phase 26) and no separate fuel slot — brewing just runs on wall time like the furnace. The brewing stand recipe substitutes a stick for the blaze rod, following the same "swap in an available material" pattern already used for the redstone torch/lever recipes.
- **Potion model** (`src/world/Brewing.ts`): a reagent slot + 3 bottle slots per stand, keyed by `"x,y,z"` in `BrewingManager` (same position-keyed block-entity pattern as `FurnaceManager`/`ChestManager`/`RedstoneManager`). Ingredient→potion mapping uses existing overworld materials: `apple→potion_healing, coal→potion_strength, gold_ingot→potion_swiftness, iron_ingot→potion_resistance`. ## Phase 29 — Weather, leaf decay & final polish ✅
- **Weather** (`main.ts`, `DayNightCycle.ts`): a simple two-state `isRaining` timer (20–60s raining, 60–180s clear), overworld-only. `DayNightCycle.raining` is a new public flag the main loop sets each tick; when true, `update()` dims the sun/ambient intensity (×0.6/×0.85) and lerps the sky/fog color 55% toward a gray-blue (`RAIN_SKY`), so rain visibly darkens the world without a separate render pass. While raining, the loop spawns falling particle streaks above the player (`Particles.rain()` — a new method spawning a disk of downward-velocity tinted points, reusing the existing pooled-particle system) and plays a soft repeating `rain_patter` sound (a new `SoundName` synthesized the same way as the existing break/place thuds — a deliberate "fake loop via repeated one-shots" simplification instead of building real looping-audio-source lifecycle management).
- **Leaf decay** (`main.ts`): breaking a `Log` calls `scheduleLeafDecayCheck(x,y,z)`, which finds every `Leaves` block within a 5-block radius sphere and queues a delayed connectivity check (0.3–0.8s). When a queued check fires, `hasNearbyLog` scans a ±4-block cube for any `Log`; if none is found the leaf decays to `Air` (via `streamer.setBlock`, so no drop — the random apple chance only applies to direct player breaks). This is a deliberate one-shot simplification of vanilla's continuous per-block BFS: it only re-checks leaves that were in range of the log that just broke, with no cascading re-checks of their neighbors, so a long orphaned branch one leaf-hop outside that radius could persist. Acceptable at this game's structure scale (single trees, no giant builds).
- **Final polish:** Ender Dragon mobs (`def.flies`) are now exempt from the distance-based mob despawn check in `MobManager.update` — without this, the dragon would vanish after wandering 80 blocks from the player, which is trivial to do by accident in the open End void.
- **Save:** no new fields — weather and pending leaf-decay timers are transient, regenerating naturally on reload; version stays at v8.
- **Verify (scripted, via `window.vox`):** directly toggled `dayNight.raining` and called `dayNight.update()` — confirmed the sky hex shifted from clear `#87ceeb` to a grayer `#6f9eb4` and back exactly on toggle. ## Fixes — water flow, crafting accuracy, mob models ✅
- **Water flow** (`main.ts`): water was previously fully static — generated once and never reacting to edits, so removing a block beside a lake left a dry hole instead of flooding. Added a delayed flood-fill: `notifyAirExposed(x,y,z)` (called from `interaction.onBlockBroken`) checks the 6 neighbors of newly-emptied Air for `Block.Water`; if found, it queues that cell (`pendingWaterFlow`) to become Water after a short delay (0.12–0.22s, for a visible "flowing" feel rather than an instant swap). When a queued cell fires, it re-validates (still Air, still has a water neighbor), converts to Water, then re-queues its own neighbors — falling straight down is unbounded (depth resets to 0, so a shaft under a lake fills all the way), while sideways spread is capped at `MAX_WATER_FLOW_DEPTH = 5` hops so a lake doesn't drain across the entire map (a deliberate simplification of vanilla's distance-from-source falloff). Overworld-only; the queue is cleared on every `teleport()` since Nether/End never place Water and the coordinates would otherwise be reinterpreted in the wrong dimension.
- **Crafting accuracy** (`Recipes.ts`): three recipes had drifted from the real game and are now fixed. **Redstone Lamp** used a leftover glass-ring placeholder from before Glowstone existed — now the real vanilla shape (Glowstone center in a plus of 4 Redstone Dust). **Enchanting Table** was missing its Diamond entirely (book + 6 obsidian, no diamond) — now book on top, Diamond centered in the obsidian middle row, obsidian across the bottom (the real shape). **Glass Bottle** was shapeless (any 3 glass anywhere) — now the real "V" shape (two top corners + one bottom-middle). Verified the old placements no longer match and the corrected ones do. Other simplified recipes (Book from 3 planks instead of paper+leather; Brewing Stand/Flint and Steel/End Portal Frame/Eye of Ender substituting materials this engine has no source for) are intentional, already-documented simplifications and were left as-is.
- **Mob models** (`Mob.ts`): heads are now a separate `THREE.Group` mounted at the neck instead of fixed to the body, so they can turn independently — idle mobs now slowly glance toward a nearby player (within 10 blocks, clamped to ±60°) and relax back to forward when they start moving, a classic "alive" Minecraft behavior. Added simple flat eye accents to every mob (the Ender Dragon's are emissive purple and glow). Pig/sheep got snout/wool-cap/ear/tail detail and a two-layer wool-over-skin look for sheep; zombie/zombie pigman got a tattered-hem torso accent; villager got a robe hem and eyes; the dragon got a chest ridge, tapered wing tips, four clawed legs, and a tail tip. Villager arms now swing opposite the legs while walking (zombie/pigman arms deliberately stay rigid in their forward reach — that's the iconic pose, not a walk cycle). Fixed a latent bug in the process: the hurt-flash emissive write touched every material unconditionally, which would have permanently erased the dragon's glowing eyes the first time it took damage — materials now store their resting emissive color and restore it instead of zeroing it.
- **Verify (scripted, via `window.vox`):** built a water pool behind a sand wall, broke the wall via `breakAt` + `notifyAirExposed`, and replicated the exact per-tick flow logic — water correctly spread sideways and stopped exactly at the depth cap (7 cells: depths 0–5). Confirmed via the crafting screen's virtual slots that the corrected Redstone Lamp/Enchanting Table/Glass Bottle placements now produce results while the old (buggy) placements no longer match. Spawned all 6 mob types and confirmed each has a head group; ran 30 manual update ticks on the dragon and confirmed its eye emissive color survived unchanged; killed all spawned mobs and confirmed `dispose()` (now traversal-based to reach meshes nested inside the head group) ran without errors. Zero TypeScript errors, zero console errors throughout.
